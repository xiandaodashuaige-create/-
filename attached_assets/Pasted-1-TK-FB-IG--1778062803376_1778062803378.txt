1. 总体结论
当前版本相比上次有明显进步，已经更接近你要的业务闭环：

TK / FB / IG:
授权账号 → 输入类目/需求 → 市场/同行分析 → 生成策略 → 生成草稿 → 客户编辑素材 → 推荐发布时间 → 排期/发布
XHS:
无需授权 → 填账号画像 → 同行/市场分析 → 伪原创图文/素材 → 推荐排期 → 客户下载后手动发布
但还不能说完全符合生产要求。当前状态我评估为：

整体符合度：约 78%

主要已经补齐：

品牌画像表 brand_profiles
品牌画像 API
品牌画像注入策略生成
FB/IG market data 从同行库聚合，不再完全 mock
XHS market data 接入公开搜索
Sora / Seedance 视频任务持久化
Autopilot 新增“编辑素材”步骤
非 XHS 平台新增授权校验
一周排期 CTA 初步接入
但仍有几个阻塞/高风险点：

批量一周排期创建的 content 没写 ownerUserId，会导致内容列表不可见或后续内容归属异常。
Autopilot 选账号时仍可能默认选中未授权账号，导致后续排期失败。
Autopilot 的“一周内容”仍是后置 CTA，不是主流程默认完整 Campaign。
图片/视频素材不是每条周计划自动生成，仍需要用户单条处理。
XHS 的 /competitors、/market-data 仍走 needs-auth guard，本质是要求创建账号画像；文案可接受，但要避免让用户理解成授权。

场景 A：TK / FB / IG 授权账户
目标
客户先授权 TK / FB / IG 账号，然后系统基于授权账号做策略、市场分析、自动排期/发布。

当前实现
相关代码：

artifacts/api-server/src/routes/oauth.ts
artifacts/xhs-tool/src/components/PlatformGuard.tsx
artifacts/api-server/src/services/publishDispatcher.ts
artifacts/api-server/src/routes/content.ts
artifacts/api-server/src/routes/schedules.ts
这次新增了前端授权拦截：

function isAccountReady(a: any): boolean {
  if (!a) return false;
  if (a.platform === "xhs") return true;
  if (a.ayrshareProfileKey && String(a.ayrshareProfileKey).length > 0) return true;
  return a.authStatus === "authorized";
}
后端排期也加了授权校验：

if (!isAccountReadyToPublish(account)) {
  res.status(400).json({
    error: "account_not_authorized",
    message: `${account.platform} 账号尚未授权，无法批量排期。请先到「账号管理」完成授权。`,
  });
  return;
}
测试结论
部分通过。

优点：

如果没有授权账号，页面会引导去授权。
如果排期时账号未授权，后端会拦截。
立即发布和定时发布都调用统一 dispatchContentToProvider。
风险：

Autopilot 里仍然这样默认选账号：

if (!stillExists) setSelectedAccountId(list[0].id);
如果用户有两个账号：

账号 A：手动添加，未授权
账号 B：OAuth 授权成功
NeedsAuthGate 只判断“是否存在任意 ready 账号”，会放行；但 Autopilot 可能默认选中账号 A。后面策略、草稿、排期会绑定 A，最终排期失败。

建议
Autopilot 应只展示/默认选择 ready accounts：

const readyAccounts = accountsQ.data?.filter(isAccountReady) ?? [];
并且：

TK / IG / FB 自动发布流只允许 ready account；
手动添加的非授权账号只能用于“策略草稿”，不能进入发布排期。
