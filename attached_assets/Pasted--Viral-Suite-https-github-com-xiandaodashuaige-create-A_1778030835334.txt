1.1 SQL 注入风险 [CRITICAL]
文件: artifacts/api-server/src/services/publishDispatcher.ts
问题描述: markPublished, markFailed, recoverStalePublishing 等函数直接使用字符串模板拼接SQL，存在SQL注入风险。
TypeScript
复制
// 问题代码 (publishDispatcher.ts: markPublished)
await db.execute(sql`
  UPDATE schedules SET status='published', remote_post_id=${remotePostId}, error_message=NULL
  WHERE id=${scheduleId}
`);
虽然使用了 sql 标签函数，但 remotePostId 和 scheduleId 是外部传入的参数。Drizzle 的 sql 模板标签确实会参数化处理，但需要确认是否所有地方都正确使用。更严重的是 db.execute(sql
awSql
awSql) 这种写法。
建议: 使用 Drizzle ORM 的 type-safe update API 替代原始SQL：
TypeScript
复制
await db.update(schedulesTable)
  .set({ status: 'published', remotePostId, errorMessage: null })
  .where(eq(schedulesTable.id, scheduleId));

2.1 publishDispatcher 的查询优化
文件: artifacts/api-server/src/services/publishDispatcher.ts

复制

四、定时发布调度器 (publishDispatcher) 分析
4.1 并发安全 [GOOD]
文件: artifacts/api-server/src/services/publishDispatcher.ts
良好的实践:
原子认领: 使用 FOR UPDATE SKIP LOCKED 防止多实例竞争
进程内 isRunning 标志: 防止单实例内重叠执行
Stale publishing 恢复: 15分钟后将卡住的 publishing 状态恢复为 pending
TypeScript
复制
// 优秀的实现
WITH due AS (
  SELECT id FROM schedules
  WHERE status = 'pending'
  AND scheduled_at <= NOW()
  AND platform IN ('tiktok','instagram','facebook')
  ORDER BY scheduled_at ASC
  LIMIT 20
  FOR UPDATE SKIP LOCKED
),
claimed AS (
  UPDATE schedules SET status = 'publishing'
  WHERE id IN (SELECT id FROM due)
  RETURNING id
)
