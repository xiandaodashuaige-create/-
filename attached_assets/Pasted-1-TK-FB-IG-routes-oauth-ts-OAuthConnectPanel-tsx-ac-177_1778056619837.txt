2. 最大的问题
P0：TK/IG/FB 现在会把“手动添加账号”当成可用账号
路径：

artifacts/xhs-tool/src/pages/accounts.tsx
artifacts/xhs-tool/src/components/PlatformGuard.tsx
现在 PlatformGuard 只判断：

accounts.length > 0
但没有判断：

authStatus === "authorized"
所以客户可以手动添加一个 TikTok / IG / FB 占位账号，然后 Autopilot 允许继续跑，最后排期发布时因为没有 OAuth token 而失败。

建议：

XHS：允许手动账号画像。
TK/IG/FB：必须要求 authStatus === "authorized" 或 ayrshareProfileKey 存在。
非授权账号只能用于“策略草稿”，不能用于“排期自动发布”。
P0：Autopilot 只生成“单条策略草稿”，不是“一周内容作战计划”
路径：

artifacts/xhs-tool/src/pages/autopilot.tsx
artifacts/api-server/src/routes/strategy.ts
现在 Autopilot 流程是：

市场数据 → 同行数据 → 生成 3 个策略 → 选一个 → approve → 创建 1 条 content → 选 1 个发布时间
但你的目标是：

市场分析 → 生成新策略 → 生成一周 7 条内容 → 每条有文案/图/视频 → 客户确认
目前“一周计划”在另一个地方：

artifacts/xhs-tool/src/pages/schedules.tsx
artifacts/api-server/src/routes/ai.ts
artifacts/api-server/src/services/planGenerator.ts
它可以生成周计划，但没有复用 Autopilot 刚生成的策略、市场洞察和客户需求上下文。

建议：
把 Autopilot 的最终输出从“1 条 content”改成：

campaign
  strategy
  marketAnalysis
  weeklyPlan[7]
    copy
    imagePrompt
    videoScript
    scheduledAt
    approvalStatus
P0：素材生成没有真正进入 TK/IG/FB 主流程
当前能力：

文案：有。
图片：有 API 和 XHS workflow，但 Autopilot 没自动调用。
视频：/api/ai/generate-video 存在，但 Autopilot / 周计划没接。
路径：

artifacts/api-server/src/routes/videoGen.ts
artifacts/api-server/src/services/videoJobs.ts
artifacts/api-server/src/services/videoPipeline.ts
注意 videoJobs.ts 还是进程内队列：

const jobs = new Map<string, VideoJob>();
const queue: string[] = [];
这意味着服务重启后视频任务状态会丢。

建议：

先生成视频脚本 / 分镜 / 字幕，不要默认直接生成成片。
客户确认后再调用 Seedance/Kling。
视频任务应入库，不能只存在内存。
P1：FB/IG 市场数据还不够真实
路径：

artifacts/api-server/src/routes/marketData.ts
当前：

// FB/IG/XHS 走 mock
res.json({ platform, keyword, region, source: "mock", items: getMockTrending(...) });
Meta Ads Library 也用的是：

const token = process.env.FACEBOOK_ACCESS_TOKEN;
不是客户授权得到的 Page token。

建议：

FB/IG 市场分析主要基于：
客户授权 Page / IG Business 的竞品抓取；
Meta Ads Library；
已添加同行库；
用户手动输入的竞品链接。
不要把 mock 数据混进 AI 决策，至少要加 sourceConfidence。
P1：客户账户定位太弱
路径：

artifacts/api-server/src/services/strategyGenerator.ts
现在账号定位主要是：

@nickname（region）
但你的场景需要更完整的客户定位：

行业类目
主营产品/服务
目标客群
价格带
地区
语言
品牌调性
禁止词
竞品偏好
促销/转化目标
内容形式偏好：图文 / 视频 / Reels / 长文
建议新增表或字段：

brand_profiles
- user_id
- platform
- account_id
- category
- products
- target_audience
- price_range
- tone
- region
- language
- conversion_goal
- forbidden_claims
然后所有策略、文案、图片、视频、周计划都引用这个 profile。

你的要求是：

小红书不需要授权，也不需要定时发布，只需要同行分析、伪原创、推荐排期表、客户自己下载发布。

当前 XHS：

/workflow 基本符合：
同行/灵感分析
内容生成
图片伪原创
下载素材
手动打开创作中心
publishDispatcher 不处理 XHS，这点正确。
但 /competitors、/market-data 被 guard="needs-auth" 包住，XHS 会要求创建账号画像。
我建议把文案改清楚：

不叫“授权小红书账号”
叫“填写小红书账号画像”
对 XHS 不应该出现“自动发布”“授权账号”措辞
XHS 的 schedule 应叫“推荐发布日历”或“手动发布提醒”，不要叫“定时发布”
4. 我建议的最终产品流程
A. TK / IG / FB 流程
1. 授权平台账号
   - TikTok OAuth / Ayrshare
   - Facebook Page OAuth
   - Instagram Business OAuth
2. 填写业务画像
   - 类目
   - 需求
   - 目标客户
   - 产品卖点
   - 地区
   - 风格
3. 市场分析
   - TikTok: TikHub / EchoTik / Kalodata / FastMoss
   - IG/FB: Meta Graph + Ads Library + 手动竞品
   - 输出：趋势、热门标签、发布时间、爆款结构、竞品样本
4. 生成 3 套策略
   - 教学型
   - 情绪共鸣型
   - 数据反差型
5. 客户选策略
6. AI 生成一周内容包
   - 每天 1-2 条
   - 每条包含：
     - 标题 / hook
     - 正文 / caption
     - hashtag
     - 图片 prompt
     - 视频脚本
     - 封面文案
     - 推荐发布时间
7. 客户逐条修改 / 批准
8. 生成最终素材
   - 图片
   - 视频
   - 字幕
   - 封面
9. 排期
   - TK/IG/FB 自动发布
   - 失败重试
   - 发布日志
B. XHS 流程
1. 填写账号画像，不授权
2. 输入类目/需求/竞品链接
3. 抓取小红书公开同行数据
4. AI 生成爆款策略
5. 生成伪原创图文
6. 客户修改
7. AI 给一周推荐排期
8. 客户下载素材包
9. 客户手动发小红书
XHS 不进入 publishDispatcher。
