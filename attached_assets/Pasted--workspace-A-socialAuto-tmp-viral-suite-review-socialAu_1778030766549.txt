关键问题
retry 逻辑实际跑不通。
路径：artifacts/api-server/src/services/publishDispatcher.ts

当前逻辑：

WHEN COALESCE(error_message, '') LIKE ${'%retry=' + MAX_RETRIES + '%'} THEN 'failed'
...
error_message = ${errorMessage + ` | retry=${Date.now()}`}
它检查的是 retry=3，但写入的是时间戳，所以失败任务可能一直回到 pending，无限重试。

多实例锁不足。
isRunning 只是进程内锁，部署多实例时仍会重复扫描。虽然 SQL 使用 FOR UPDATE SKIP LOCKED，但需要确认事务边界；当前 db.execute 单语句 claim 是好的，但 dispatcher 级别没有全局 advisory lock。

Ayrshare profile key 没有传入。
row.ayrshare_profile_key 只用于判断 useAyrshare，没有传给 publishToSocial。

发布结果状态回查不足。
TikTok direct 返回 publish_id 后没有持续 poll；Ayrshare 也没有定时 status reconcile。

修复建议
给 schedules 增加 retry_count integer，不要把 retry count 塞进 error_message。
markFailed 改为：
retry_count < MAX_RETRIES → pending
retry_count >= MAX_RETRIES → failed
