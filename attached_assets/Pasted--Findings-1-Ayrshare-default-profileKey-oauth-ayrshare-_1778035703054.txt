主要问题 / Findings
1. 🔴 Ayrshare "default" profileKey 可能导致自动发布失败
/oauth/ayrshare/sync 会把 Ayrshare 账号写成：

.set({ ayrshareProfileKey: "default" })
路径：artifacts/api-server/src/routes/oauth.ts:257-260

但新版 publishDispatcher 又会把它作为真实 profile key 传给 Ayrshare：

profileKey: row.ayrshare_profile_key ?? undefined,
路径：artifacts/api-server/src/services/publishDispatcher.ts:142-143

而 Ayrshare client 会直接发 header：

if (profileKey) headers["Profile-Key"] = profileKey;
路径：artifacts/api-server/src/lib/oauth/ayrshare.ts:11-17

风险： 如果 Ayrshare 主 profile 不接受 Profile-Key: default，原本能发布的默认 profile 会开始失败。

建议：

不要用 "default" 当真实 profile key。
可改为：
ayrshareProfileKey = null 表示默认 profile；
新增 publishVia = "ayrshare" 或 authProvider = "ayrshare" 字段表示走 Ayrshare。
临时修复：
profileKey:
  row.ayrshare_profile_key && row.ayrshare_profile_key !== "default"
    ? row.ayrshare_profile_key
    : undefined

5. 🟡 账号删除仍会产生隐藏内容
content.accountId 仍是：

accountId: integer("account_id").references(() => accountsTable.id, { onDelete: "set null" })
路径：lib/db/src/schema/content.ts:8

而 /content 列表依赖 INNER JOIN accounts 做租户过滤。删除账号后，内容会变成 account_id = null，从 UI/API 列表消失。

虽然这次修复了 strategy/:id/approve 创建孤儿内容的问题，但账号删除路径仍可能制造孤儿内容。
