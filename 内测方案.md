# 小红书AI工具 — 内测方案

## 一、内测前你需要完成的准备工作

### 1. GitHub 部署准备

```bash
# 步骤1：在 GitHub 创建新仓库（建议私有仓库）
# 仓库名建议：xhs-ai-tool

# 步骤2：解压代码包
tar xzf xhs-ai-tool.tar.gz -C xhs-ai-tool
cd xhs-ai-tool

# 步骤3：初始化 Git 并推送
git init
git add .
git commit -m "Initial commit: XHS AI Tool v1.0"
git remote add origin https://github.com/你的用户名/xhs-ai-tool.git
git branch -M main
git push -u origin main
```

### 2. 服务器部署所需环境变量

在你的部署平台（Vercel / Railway / 自建服务器）设置以下环境变量：

| 变量名 | 说明 | 获取方式 |
|--------|------|----------|
| `DATABASE_URL` | PostgreSQL 数据库连接串 | 使用 Supabase / Neon / Railway 等免费 PostgreSQL |
| `CLERK_SECRET_KEY` | Clerk 后端密钥 | [clerk.com](https://clerk.com) → 你的应用 → API Keys |
| `CLERK_PUBLISHABLE_KEY` | Clerk 前端公钥 | 同上 |
| `VITE_CLERK_PUBLISHABLE_KEY` | 前端 Clerk 公钥（与上面相同的值） | 同上 |
| `SESSION_SECRET` | 会话加密密钥 | 自己生成一个随机字符串（32位以上） |
| `AI_INTEGRATIONS_OPENAI_API_KEY` | OpenAI API 密钥 | [platform.openai.com](https://platform.openai.com) |
| `AI_INTEGRATIONS_OPENAI_BASE_URL` | OpenAI API 地址 | 默认 `https://api.openai.com/v1` |
| `NODE_ENV` | 环境标识 | 生产环境设为 `production` |
| `PORT` | 服务端口 | 根据平台设置，通常 `8080` 或 `3000` |

### 3. 数据库初始化

```bash
# 安装依赖
pnpm install

# 初始化数据库表结构
cd lib/db
pnpm drizzle-kit push
```

### 4. Clerk 配置（用户登录系统）

1. 登录 [clerk.com](https://dashboard.clerk.com)
2. 进入你的应用设置
3. **开启 Google 社交登录**：User & Authentication → Social Connections → 开启 Google
4. **生产环境**：需要创建生产实例（Production Instance），获取生产环境的 Key
5. **自定义域名**：Settings → Domains → 添加你的正式域名

### 5. 把自己设为管理员

首次登录系统后，在数据库中执行：

```sql
-- 把你的账号设为管理员（用你的注册邮箱替换）
UPDATE users SET role = 'admin', plan = 'pro', credits = 99999
WHERE email = '你的邮箱@example.com';
```

## 二、内测流程

### 阶段 1：自测（1-2天）

你自己走完以下完整流程，确认所有功能正常：

- [ ] 注册/登录（确认 Google 登录可用）
- [ ] 新手引导弹窗正常显示
- [ ] 添加小红书账号（SG/HK/MY）
- [ ] 完整发布向导 5 步流程：
  - [ ] 选择账号
  - [ ] AI竞品分析（消耗 5 积分）
  - [ ] AI改写 + 生成标题/标签/配图
  - [ ] 敏感词检测 + 预览
  - [ ] 发布
- [ ] AI助手对话正常
- [ ] 素材上传/管理
- [ ] 管理后台：查看用户、充值积分、改角色/套餐
- [ ] 中英文切换

### 阶段 2：邀请内测用户（3-7天）

1. 准备 5-10 个内测用户
2. 发送内测邀请，包含：
   - 网站链接
   - 简要操作说明
   - 反馈收集渠道（微信群 / 问卷）

3. 内测用户的默认状态：
   - 自动获得 **20 积分**（够体验一次完整发布流程）
   - 默认 **免费版**
   - 默认 **1 个小红书账号**

4. 你作为管理员可以：
   - 在 `/admin` 后台随时给用户充值积分
   - 升级用户套餐（free → starter → pro）
   - 查看每个用户的积分消耗记录

### 阶段 3：收集反馈并迭代

重点关注：
- 注册登录是否顺畅
- AI 生成质量是否满意
- 积分消耗是否合理
- 有无功能缺失或 Bug
- 价格是否可接受

## 三、推荐部署方案对比

| 方案 | 优点 | 缺点 | 费用 |
|------|------|------|------|
| **Replit 直接发布** | 零配置，一键发布 | 受 Replit 平台限制 | 包含在 Replit 订阅中 |
| **Vercel + Supabase** | 免费额度大，全球 CDN | 需要分离前后端部署 | 免费起步 |
| **Railway** | 支持全栈一键部署 | 免费额度有限 | $5/月起 |
| **自建 VPS** | 完全控制 | 需要自己维护 | $5-20/月 |

### 最简单的方式：直接在 Replit 发布

如果你想最快上线内测，可以直接用 Replit 的发布功能：
- 一键部署，自动配置 HTTPS
- 自带 `.replit.app` 域名
- 支持绑定自定义域名
- 数据库、存储、环境变量全部自动配置

## 五、技术架构概览

```
前端：React + Vite + TypeScript + TailwindCSS + shadcn/ui
后端：Express + TypeScript + Drizzle ORM
数据库：PostgreSQL
用户认证：Clerk（支持邮箱 + Google 登录）
AI服务：OpenAI GPT-4o-mini（文本） + DALL-E 3（图片）
文件存储：Replit Object Storage / 可替换为 S3
```
